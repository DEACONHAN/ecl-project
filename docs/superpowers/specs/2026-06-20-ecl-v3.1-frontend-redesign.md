# ECL v3.1 前端重构 & 后端补齐 — 设计说明书

> **日期**：2026-06-20
> **依据**：design-variants-cp/ V1 设计稿
> **范围**：前端全量 V1 对齐 + 后端方案管理 & 参数层补齐

---

## 1. 目标与范围

### 1.1 目标

将 ecl-frontend 从当前 Ant Design 默认暗色布局，重构为 design-variants-cp/V1 设计稿的 **Refined Enterprise / Clean Professional** 风格，同时确保后端 API 能完整支撑前端交互。

### 1.2 范围

| 层级 | 包含 | 不包含 |
|------|------|--------|
| 前端 | 全局布局、方案列表、方案总览、方案对比、风险分组、阶段划分、PD/LGD/CCF参数、管理层叠加 | 跑批监控（仪表盘）、试算中心 |
| 后端 | 方案管理 API、6个参数模块 API 的查漏补缺 | 计算引擎、报表层、试算执行 API |

### 1.3 设计稿映射

| 设计稿 (design-variants-cp) | 前端页面 | 路由 |
|---|---|---|
| index.html (App Shell) | MainLayout | / |
| v1-scheme-list.html | SchemeList | /schemes |
| v1-scheme-overview.html | SchemeOverview | /schemes/:id |
| v1-risk-group-config.html | RiskGroupConfig | /parameters/risk-groups |
| v1-stage-config.html | StageConfig | /parameters/stage |
| v1-pd-config.html | PdConfig | /parameters/pd |

## 3. 布局架构

### 3.1 App Shell（MainLayout）

```
┌──────────────────────────────────────────────────┐
│ Top Header (52px)                                  │
│ [ECL Logo] [跑批监控] [方案列表] [试算中心]          │
├────────────┬─────────────────────────────────────┤
│ Sidebar    │ Content Area                         │
│ (224px)    │ (flex:1, overflow:auto)              │
│            │                                      │
│ 方案编码    │   <Outlet />                         │
│ 方案名称    │                                      │
│ 状态徽标    │                                      │
│ ─────────  │                                      │
│ 方案总览    │                                      │
│ 风险分组    │                                      │
│ 阶段划分    │                                      │
│ PD 参数    │                                      │
│ LGD 参数   │                                      │
│ CCF 参数   │                                      │
│ 管理层叠加  │                                      │
└────────────┴─────────────────────────────────────┘
```

- **Sidebar 显示条件**：仅在进入方案上下文时显示（通过 `outletContext` 或路由状态控制）
- **顶级页面**（方案列表、方案对比、跑批监控占位、试算中心占位）：Sidebar 隐藏
- **方案子页面**（总览、参数配置）：Sidebar 显示，含面包屑导航

### 3.2 路由设计

```
/                          → Navigate to /schemes
/schemes                   → SchemeList
/schemes/compare           → SchemeCompare
/schemes/:id               → SchemeOverview (含 sidebar context)
/parameters/risk-groups    → RiskGroupConfig (需要 schemeId query param)
/parameters/stage          → StageConfig (需要 schemeId)
/parameters/pd             → PdConfig (需要 schemeId)
/parameters/lgd            → LgdConfig (需要 schemeId)
/parameters/ccf            → CcfConfig (需要 schemeId)
/parameters/overlay        → OverlayConfig (需要 schemeId)
```

参数配置页通过 URL query param `?schemeId=xxx` 获取方案上下文。如果缺失，页面内部显示方案选择器。

## 5. 后端补齐清单

### 5.1 需新增的 API

| 模块 | 端点 | 说明 |
|------|------|------|
| Scheme | `GET /api/v1/schemes/{id}/summary` | 返回方案总览所需的各模块统计（风险分组数、PD曲线数等） |

### 5.2 需确认/可能调整的 API

| 模块 | 端点 | 问题 |
|------|------|------|
| Stage | CRUD endpoints | 确认 riskGroupId 参数支持 |
| PD | Curve endpoints | 确认是否需要 riskGroupId 过滤 |
| LGD | Curve endpoints | 确认是否需要 riskGroupId 过滤 |
| RiskGroup | Details update | 确认批量更新 details 的 endpoint |

### 5.3 后端改动策略

优先前端开发，后端只做必要的最小改动：
1. 先在 `SchemeVO` 中增加统计字段（前端可先 mock）
2. 确认各参数模块 API 的 filter 参数是否满足前端需求
3. 仅在前端联调受阻时才修改后端 API

## 6. 组件拆分

### 6.1 共享组件

| 组件 | 用途 | 出现页面 |
|------|------|----------|
| `GlobalContext` | 顶部 Context Bar | 所有页面 |
| `Panel` | 卡片容器（header + body） | 所有页面 |
| `StatusTag` | 状态徽标 | SchemeList, SchemeOverview |
| `GroupSelector` | 风险分组选择器（Chips） | Stage, PD, LGD, CCF |
| `PageHeader` | 页面标题区 | 所有页面 |
| `TabNav` | 下划线 Tab 切换 | Stage, LGD |
| `ConfirmModal` | 确认弹窗 | 所有删除操作 |

### 6.2 页面内组件

每个页面内部的复杂逻辑（如条件编辑器、矩阵热力图等）保持页面内组件，不提升到全局，避免过早抽象。
