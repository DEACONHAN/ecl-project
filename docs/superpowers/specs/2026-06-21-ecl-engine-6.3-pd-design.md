## 1. 目标

实现 PD 计算引擎（6.3），从 PD 曲线缓存中按 (groupId, ratingCode, scenarioId) 查 PD 值，加权计算 12M PD，并按借据所处阶段转换为最终 PD（Stage 1=12M PD，Stage 2=存续期 PD，Stage 3=1.0）。PD 无兜底——查不到即标记阻断。

## 5. PdEngine

### 5.1 类定义

```java
@Component
@RequiredArgsConstructor
public class PdEngine implements EclEngine {

    private final PdScenarioMapper scenarioMapper;
    private final PdCurveMapper curveMapper;

    @Override
    public EngineType getType() { return EngineType.PD; }
}
```

### 5.2 execute() 流程

```
1. 加载情景列表 List<PdScenario>（按 schemeId，包含 scenarioId + weight）
2. 构建 PD 曲线缓存 Map<String, Double> key = "groupId|ratingCode|scenarioId"
3. 逐客户逐借据：
   a. 取 groupId, ratingCode, stageResult
   b. 遍历各情景：
      - 查缓存 → pdValue
      - null → pdException = "ECL_001"，跳过该情景
      - 计算 weightedPd = pdValue × weight，加入 pdDetails
   c. pd12m = Σ weightedPd
   d. convertPdByStage(pd12m, stage, maturityDate, calcDate) → pdLifetime
```

### 5.3 Stage 转换

| Stage | pdLifetime |
|-------|-----------|
| STAGE_1 | pd12m（直接使用） |
| STAGE_2 | 1 - (1 - pd12m)^T，T = max(剩余月数/12, 1) |
| STAGE_3 | 1.0（全额覆盖） |

### 5.4 阻断策略

- PD 曲线缺值 → `pdException = "ECL_001"`，**不抛异常**，标记后继续处理其他借据
- 情景列表为空 → 所有借据标记 ECL_001

## 6. 测试场景

| 场景 | 预期 |
|------|------|
| 情景列表为空 | 所有借据 pdException="ECL_001" |
| 单情景 + 正常 PD 值 | pd12m = pdValue × weight，pdDetails 含 1 条 |
| 三情景加权 | pd12m = Σ(pdValue_i × weight_i) |
| Stage 1 | pdLifetime = pd12m |
| Stage 2 | pdLifetime = 1 - (1-pd12m)^T |
| Stage 3 | pdLifetime = 1.0 |
| 某情景 PD 缺值 | 该情景跳过，pdException 标记 |
| 评级代码无匹配 | 所有情景缺值，pdException="ECL_001" |
