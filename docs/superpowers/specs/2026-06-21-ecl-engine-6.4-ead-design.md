## 1. 目标

实现 EAD 计算引擎（6.4），计算每笔借据的表内 EAD（未偿余额+应计利息）和表外 EAD（未提取额度×CCF），汇总为总 EAD。CCF 缺值使用方案 defaultCcf。

**当前简化**：借据级计算，不涉及跨借据额度分配和押品池。CustomerContext 不扩展。

## 4. EadEngine

### 类定义

```java
@Component
@RequiredArgsConstructor
public class EadEngine implements EclEngine {
    private final CcfCurveMapper ccfCurveMapper;
    @Override public EngineType getType() { return EngineType.EAD; }
}
```

### 核心公式

```
onBsEad  = outstandingBalance + accruedInterest
offBsEad = max(0, totalLimit - outstandingBalance) × CCF
totalEad = onBsEad + offBsEad
```

### CCF 查表

- 缓存 key = `productType + "|" + commitmentType`
- CCF 缺值 → `defaultCcf`（从 ctx 获取，不阻断）

### 测试场景

| 场景 | 预期 |
|------|------|
| 仅表内（无表外额度） | offBsEad=0, totalEad=onBsEad |
| 表内+表外 | totalEad = onBsEad + offBsEad |
| CCF 缺值 | 使用 defaultCcf |
| CCF 曲线为空 | 所有借据用 defaultCcf |
| 全额提取（balance=limit） | offBsEad=0 |
